YPTO / ProRail via Ordina Belgium N.V.

Spearheaded the GPRi application — a competency-based examination platform for train operators. Led a cross-functional team of 3 .NET backend devs, an analyst, and a UX/UI designer. Migrated legacy Access application to modern Mendix architecture.

Mendix REST API SSO/SAML Postman Adobe XD UX/UI
◈ SITUATION

As Lead Developer, worked with a team of 3 .NET backend developers, an analyst and a UX/UI designer on a project called GPRi. This application is used to test the theoretical knowledge of train operators based on their competencies. The existing application GPR was built in Access and needed to be rebuilt in Mendix — more user-friendly, better designed, and better engineered.

◈ MISSION TASKS

Initially responsible for building the frontend in Mendix. Promoted to Lead Developer: responsible for full frontend development; advised backend team on optimizations, REST service quality, and data structure improvements; guided UX/UI designer on Mendix-aligned designs; reviewed analyst documents with feedback on optimizations, ambiguities, and new possibilities. Worked closely with the Product Owner from day one. Proposed creating instructional videos for end-user training.

◈ RESULT

Administrators can create questions, answers, and exams linked to specific competencies. Exam type determines the workflow: permanent training exams are assigned by instructors to train operators; quarterly exams auto-assign questions based on operator competencies. Train operators can complete and submit their exams digitally.

MENDIX DEVELOPER

Continental via Ordina Belgium N.V.

Built the First Production Run application — a workflow system for registering production process changes. Implemented automated email notifications, PDF generation, and SSO authentication via CLAP StarterApp integration.

Mendix SSO Design Thinking Agile
◈ SITUATION

Continental Belgium uses a Lotus Notes application for registering workflows around production process changes. Since Lotus Notes is being phased out at Continental, this process needs to be migrated to a new application — with improved usability so that working with the new tool is faster.

◈ MISSION TASKS

Using Design Thinking, built a Mendix application for creating new First Production Runs. Automated email notifications with direct links to the relevant production run data. Integrated document generation capabilities. Connected with the CLAP StarterApp for SSO-based user management. Applied an Agile approach: iteratively building increments to deliver full client satisfaction rapidly.

◈ RESULT

A responsive web application where different departments register their production run processes. Five different process forms available (including valves and sensors). Each department has access to specific form sections. Automated emails keep the right people informed at the right moments. PDF generation of all completed information is available.

Built a micro-app architecture for better maintenance and easier future migration. Responsibilities: Security, SAML SSO authentication, automated document creation (PDF/Excel/CSV), automated mailing, data management, REST API integration, automatic events, database migration, and logging.

◈ RESULT

Successfully delivered the Medicert application within the project timeline. The team built a complete Mendix app using micro-app architecture within one month, enabling better maintainability and easier future version migration.

AUG 2020 — OCT 2020 ● COMPLETED

MENDIX DEVELOPER

FOD MOB — Gordeldracht via Ordina Belgium

Rebuilt a legacy Access application for seatbelt exemption management. Citizens authenticate via e-ID, submit requests reviewed by administrative officers. Implemented full SAML SSO, document generation, automated mailing, REST APIs, and audit trail logging.

Mendix e-ID Auth SAML AuditTrail Design Thinking
◈ SITUATION

Worked in a team of four (architect, Mendix Expert, Mendix Rapid developer) on the Minerva project. Gordeldracht was a legacy Access application that had to be rebuilt in Mendix because it was no longer maintainable.

◈ MISSION TASKS

Citizens authenticate via e-ID (electronic identity card) and submit seatbelt exemption requests — also possible for others using national registry number. Administrative officers review requests based on attached medical certificates. Built: Security, SAML SSO authentication, automated document creation (PDF/Excel/CSV), automated mailing, data management, micro-app architecture, REST APIs for inter-module data exchange, automatic events, database migration, AuditTrail, and logging. Applied Agile methodology with Design Thinking.

◈ RESULT

Application is in production. Administrative assistants at the Federal Public Service for Mobility (FOD MOB) can enter and generate exemptions. Citizens submit documents to FOD MOB for processing by administrative assistants. Future expansion planned for direct digital document submission by citizens.
